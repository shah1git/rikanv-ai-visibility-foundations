# Article Quality Checklist

- [ ] Есть цель статьи.
- [ ] Есть конкретная аудитория.
- [ ] Есть content task.
- [ ] Есть claims used.
- [ ] Есть sources used.
- [ ] Нет выдуманных характеристик.
- [ ] Все TODO видимы и не выглядят как факты.
- [ ] Есть FAQ, если тема предполагает вопросы.
- [ ] Есть блок “кому подходит / кому не подходит”, если это buyer guide или сравнение.
- [ ] Есть дата обновления.
- [ ] Понятна структура для человека.
- [ ] Понятна структура для AI: заголовки, таблицы, FAQ, определения.
- [ ] Есть внутренние ссылки на product profiles, тесты или FAQ.
- [ ] Нет рекламной воды.
- [ ] Нет утверждения “лучший” без verified evidence.
- [ ] Есть review status.
