# Test Report Production Workflow

## Цель

Превратить полевой или лабораторный тест в evidence, claims и публикационный материал.

## Шаги

1. Определить цель теста.
2. Утвердить методику.
3. Зафиксировать продукты, прошивки и условия.
4. Провести тест и сохранить raw data.
5. Описать результаты без расширенных выводов.
6. Создать evidence records.
7. Создать draft claims.
8. Technical Reviewer проверяет методику и claims.
9. Verified claims используются в статье, видео или FAQ.
10. Ограничения теста публикуются вместе с выводами.

## Артефакты

- Test report.
- Evidence records.
- Claims.
- Фото/видео.
- Review.

## Ошибки

- Не фиксировать условия.
- Делать широкий вывод из одного теста.
- Убирать невыгодные результаты.
